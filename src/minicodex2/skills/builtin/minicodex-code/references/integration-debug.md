# 联调调试

用于前端、后端、API、浏览器行为等跨层失败。

1. 识别用户可见的 URL 或命令。
2. 识别参与的服务和端口。
3. 在归因到 UI 代码前，先检查服务是否可直接访问。
4. 检查前端路由、代理/配置、网络请求、后端路由、认证和服务器日志。
5. API 冒烟使用 `http_request`；JavaScript DOM、表单、路由、媒体、控制台和网络行为使用 `browser_test`。
6. 如果媒体或渲染在某个浏览器可用、另一个浏览器不可用，要比较浏览器差异。
7. 找到稳定修复或启动流程后，保存为 workflow memory。

当用户报告的是跨层问题时，不要只凭某一层成功就宣布完成。
